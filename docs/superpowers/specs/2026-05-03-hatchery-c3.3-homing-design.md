# Hatchery vs Wild C3.3 — Homing Precision

**Date:** 2026-05-03
**Owner:** @razinkele

This is the **final tier** of hatchery-vs-wild origin support. Tier
sequence:

- C1 (shipped, PR #3 merged 2026-05-02, commit `5791802`, v1.7.4) —
  tag-only origin int8 column on AgentPool + 7-touch OutputLogger.
- C2 (shipped, PR #4 merged 2026-05-02, commit `8a9192c`, v1.7.4) —
  `activity_by_behavior` parameter divergence (+25% on RANDOM/UPSTREAM
  for hatchery); HatcheryDispatch bundle + Landscape injection.
- C3.1 (shipped, PR #5 merged 2026-05-02, commit `0284c45`, v1.7.5) —
  pre-spawn skip probability divergence (`p_skip = 0.3` for hatchery).
- C3.2 (shipped, PR #6 merged 2026-05-03, commit `17ae6b5`, v1.7.6) —
  sea-age trinomial sampling at IntroductionEvent + PatchIntroductionEvent.
- **C3.3 (this spec)** — homing precision divergence: when an agent
  first enters a delta branch, draw the chosen branch via origin-aware
  homing-precision; teleport to the chosen branch's seaward entry cell
  on stray. ~1 week.

C3.3 closes the four-tier hatchery-vs-wild architecture: C1 tags
origin; C2 routes parameter divergence at the bioenergetics layer;
C3.1 routes parameter divergence at the reproduction layer; C3.2
routes parameter divergence at the introduction layer (per-cohort
sea-age); C3.3 routes parameter divergence at the **return-migration
layer** (delta-branch homing fidelity).

## Why now

The Lithuanian hatchery programme releases smolts at the river-mouth
or delta-branch level rather than at upstream tributaries. This
release timing skips the parr-stage natal-stream imprinting window
that normally produces high homing fidelity in wild-origin Atlantic
salmon. The empirical signal across the Baltic is consistent:
**hatchery-released-as-smolts salmon have markedly higher straying
rates at the delta-branch level than wild-origin salmon**. C3.3 is
the architectural enabler for measuring branch-distribution contrast
between wild and hatchery cohorts in mixed-origin scenarios — the
C2 + C3.1 + C3.2 differences are between-agent biology, while C3.3
is the first tier that produces a measurable spatial-distribution
divergence at the population level.

## Scientific basis

**Primary anchor (Baltic-specific, hatchery-vs-wild straying):**
Vasemägi, Gross & Paaver (2005), *Heredity* 95(1):76–83,
doi:10.1038/sj.hdy.6800693 — *"Extensive immigration from
compensatory hatchery releases into wild Atlantic salmon population
in the Baltic sea: spatio-temporal analysis over 18 years"*. Genetic
mixed-stock analysis of Vindelälven (Baltic) over 1985–2003.
Verbatim: *"66% of fin-damaged spawners (n = 181) caught in the
Ume/Vindelälven during 1997-2003 originated from the hatcheries."*
Maximum-likelihood immigration rate **0.068 (95% CI 0.021–0.128)**
baseline; peak **0.249 (95% CI 0.106–0.419)** during 1993–2000.
Verbatim on differential: *"For hatchery-reared Atlantic salmon,
**more than twice the straying rate of wild conspecifics** has been
reported (Jonsson et al, 2003)."* DOI verified via scite.

**Secondary anchor (release-site fidelity validates the model):**
Insulander & Ragnarsson (2001), *Fisheries Management and Ecology*
8(1):61–67, doi:10.1046/j.1365-2400.2001.00177.x — *"Homing patterns
of Baltic salmon, Salmo salar L., from smolts released from two
hatcheries in the River Dalälven, Sweden"*. Two hatcheries 600 m
apart on the same river. Salmon homed back to their RELEASE point.
**Validates** C3.3's mechanistic assumption that hatchery agents'
homing target is `natal_reach_id` (set at introduction = release
point per C1's tagging). DOI verified.

**Mechanistic anchor (imprinting timing):** Källo, Baktoft &
Kristensen (2022), *ICES Journal of Marine Science* 79(5):1539–1547,
doi:10.1093/icesjms/fsac079 — *"High prevalence of straying in a
wild brown trout (Salmo trutta) population in a fjord system"*.
PIT-tagged 21,538 juveniles. Verbatim: *"parr had lower likelihood
to stray compared to pre-smolts and smolts"*. Mechanistic support
for the smolt-stage release pattern producing higher straying. DOI
verified.

**Cross-tier anchor:** Jokikokko, Kallio-Nyberg & Saloniemi (2006),
*Journal of Fish Biology* 68(2):430–442,
doi:10.1111/j.0022-1112.2006.00892.x — *"The survival of semi-wild,
wild and hatchery-reared Atlantic salmon smolts of the Simojoki
River in the Baltic Sea"*. Same population as C3.2's Jokikokko 2004
anchor. Recapture rates: semi-wild parr-released **1.0–13.1%**
(similar to wild **1.7–17.0%**); hatchery-reared smolts **1.3–6.3%**
(2–3× lower). DOI verified.

**Generation-level applicability:** Žeimena/Simnas programme since
1997 (~29 years; ~4–7 salmon generations) uses smolt-stage release.
Same release-practice regime as Vasemägi 2005 Vindelälven sources.
Direct Lithuanian measurement not yet available; defaults are
calibration-grade.

  resulting cluster within ~5 steps. A future tier could pick the
  seaward-most cell (smallest cell-to-lagoon distance, or first cell

  males straying more than females. Defer with sex tracking.

- **Multi-spawning re-homing.** First-touch sticky semantics.
- **Origin inheritance** — per C1 scope-OUT.
- **Cross-population transfer alignment.** When `SwitchPopulationEvent`

  `landscape["hatchery_dispatch"]` is None, the agent gets a logged

  fix is multi-population alignment in `network.py`.
- **Per-cohort vectorised dispatch optimisation.** The per-agent
  loop is acceptable at expected scale (~10000 simultaneous arrivals
  worst-case). If profiling shows >50ms in this path, vectorise via
  precomputed `(branches_array, stray_probs_array)` per `natal_rid`.

## Sentinel & state-consistency discipline

`pool.exit_branch_id == -1` continues to mean "agent has not yet
touched a delta branch reach". Once homing fires, `exit_branch_id`
becomes either the natal branch (with probability `p_homing`) or a
stray branch. Sticky: never reset.

**State-update atomicity (load-bearing):** the homing draw produces
a `(staged_rid, staged_cell)` pair PER AGENT. The dispatch builds
the full per-agent staged arrays in a Python loop, then commits
both writes vectorised AT END:

```
for i in target_indices:
    staged_rid[k] = drawn_rid_for_agent_i  # in local array, NOT pool
    staged_cell[k] = entry_cell_for_drawn_rid  # NOT pool either
# After loop: vectorised commit (atomic per-agent)
pool.exit_branch_id[target_indices] = staged_rid
teleport_mask = staged_cell != -1  # -1 means "no teleport"
pool.tri_idx[target_indices[teleport_mask]] = staged_cell[teleport_mask]
```

Rationale: if `_branch_entry_cell` raises mid-loop (mesh corruption
edge case), no pool field has been mutated. **A mid-step raise that
half-mutates `exit_branch_id` while leaving `tri_idx` in the wrong
branch is a categorically worse failure than a clean raise, because
the simulation continues with desynced state.** Test 11 locks this
invariant by injecting a controlled exception in the mid-loop and
asserting the pool is unchanged.

**Post-teleport drift is permitted by design.** The branch entry
cell may sit on an H3 mesh boundary; subsequent UPSTREAM movement
may carry the agent into a neighbour cell whose `reach_id` differs
from `exit_branch_id`. This is intentional — `exit_branch_id`
records the homing **decision**, NOT the agent's instantaneous
location. Do NOT re-tag on subsequent branch crossings (would
violate the empirical "% returners in wrong branch" metric, which
counts decisions, not crossings). The OutputLogger schema doc
(C3.2's `docs/scientific-foundations-hatchery-wild.md` §3 OutputLogger
table) gets a one-line note.

**`AgentPool.exit_branch_id` semantic widening:** the field's
ARRAY_FIELDS comment in `salmon_ibm/agents.py` updated from "first
delta-branch reach_id touched; sticky; -1 if never" to "delta-branch
homing decision: -1 = untagged, natal_rid = homed, ≠ natal_rid =
strayed (set at first-touch via origin-aware draw); sticky".

## Divergent values

```python
# Wild baseline (NEW C3.3 field):
homing_precision = 0.95

# Hatchery (full-replacement scalar override):
homing_precision = 0.65
```

**Wild baseline:** 0.95 anchored to Vasemägi 2005's "more than twice
the straying rate" comparison implying wild straying ~5%. Insulander
2001 confirms high wild fidelity when imprinting is intact.

**Hatchery shift:** 0.65 (~35% straying) midpoint of Baltic empirical
range:
- Vasemägi 2005 baseline 0.068 (~7% straying for dominant hatchery
  source) — lower bound.
- Vasemägi 2005 peak 0.249 (~25%) during M74 outbreaks — upper bound
  for individual stocks.
- Källo 2022 wild brown trout 36% straying — supports up to ~0.4
  for poorly-imprinted populations.
- Effective range: ~10–35% straying, midpoint ~22% → homing ~0.78.
  Round to **0.65** as conservative-to-divergence (more contrast =
  more signal in cohort-tracking simulations).

**Effect size at default 0.65** (math verified):
For an agent with natal=Atmata: hatchery branch distribution =
- Atmata: 0.65 (homing succeeds)
- Skirvyte: 0.35 × (0.51 / (0.51 + 0.22)) = 0.35 × 0.6986 = **0.244**
- Gilija: 0.35 × (0.22 / (0.51 + 0.22)) = 0.35 × 0.3014 = **0.106**

Wild for the same agent: Atmata 0.95, Skirvyte 0.035, Gilija 0.015.

**Calibration sensitivity:** sweep hatchery `homing_precision ∈ {0.45,
0.55, 0.65, 0.75}`. Cap **0.85**. Wild 0.95 NOT swept.

Helper 1 — `_branch_entry_cell`:
```python
def _branch_entry_cell(mesh, branch_rid: int) -> int:
    """Return a deterministic cell of branch_rid for teleport target.

    NOTE: Returns the lowest-index cell with reach_id == branch_rid.
    This is a layout artifact, NOT the seaward-most cell. Acceptable
    because UPSTREAM movement disperses the post-teleport cluster
    within ~5 steps. Hydrologically-correct entry-cell selection is
    deferred to a future tier (see C3.3 spec out-of-scope).

    Returns -1 if no cells match (caller MUST treat -1 as "skip
    teleport, do not mutate pool fields").
    """
    cache = getattr(mesh, "_branch_entry_cells", None)
    if cache is None:
        cache = {}
        mesh._branch_entry_cells = cache
    if branch_rid not in cache:
        matches = np.where(mesh.reach_id == branch_rid)[0]
        cache[branch_rid] = int(matches.min()) if len(matches) > 0 else -1
    return cache[branch_rid]
```

Helper 2 — `branch_stray_weight`:
```python
def branch_stray_weight(branch_rid: int, mesh) -> float:
    """Look up BRANCH_FRACTIONS by branch_rid, NOT by name.

    BRANCH_FRACTIONS is name-keyed (Skirvyte/Atmata/Gilija). This
    wraps the int→str→float chain so dispatch call sites are
    int-keyed only. Future tributary-tier topology changes that
    add/remove branches localise the breakage to this helper.
    """
    return BRANCH_FRACTIONS[mesh.reach_names[branch_rid]]
```

Replace `update_exit_branch_id` (replace, not append):
```python
def update_exit_branch_id(pool, mesh, *, landscape=None) -> None:
    """First-touch tagging of pool.exit_branch_id by delta branch.

    Two paths:

    (A) PASSIVE (legacy, pre-C3.3 behavior). Triggered when
        landscape is None OR species_config is missing/non-Baltic
        OR agent's natal_reach_id is not a delta branch. Records
        the cell-they-happen-to-be-in via _first_touch_passive_tag.

    (B) HOMING-BIASED (C3.3). Triggered when landscape has a
        Baltic species_config AND the agent's natal_reach_id IS a
        delta branch. Draws the chosen branch via origin-aware
        homing-precision; teleports to chosen branch's entry cell
        on stray. Atomicity: stages decisions, commits at end of
        loop (no mid-loop pool mutation).

    Spec: docs/superpowers/specs/2026-05-03-hatchery-c3.3-homing-design.md
    """
    if not getattr(mesh, "reach_names", None):
        return  # legacy mesh fallback (TriMesh / HexMesh)
    branch_rids = _branch_reach_ids(mesh)
    if len(branch_rids) == 0:
        return

    # Prelude (mirrors pre-C3.3 logic).
    tri = pool.tri_idx
    safe_tri = np.where(tri >= 0, tri, 0)
    cur_reach = mesh.reach_id[safe_tri]
    is_branch = np.isin(cur_reach, branch_rids)
    on_mesh = tri >= 0
    untagged = pool.exit_branch_id == -1
    target = is_branch & untagged & pool.alive & on_mesh
    if not target.any():
        return
    target_indices = np.where(target)[0]

    # Path (A): no Baltic species_config OR no landscape — passive tag.
    species_cfg = landscape.get("species_config") if landscape else None
    is_baltic = (
        species_cfg is not None
        and isinstance(species_cfg.wild, BalticBioParams)
    )
    if not is_baltic:
        _first_touch_passive_tag(pool, target_indices, cur_reach)
        return

    # Path (B): Baltic — origin-aware homing draw with atomicity.
    hd = landscape.get("hatchery_dispatch")
    rng = landscape.get("rng", np.random.default_rng())
    branch_rids_set = set(int(r) for r in branch_rids)
    n_targets = len(target_indices)
    staged_rid = np.empty(n_targets, dtype=np.int8)
    staged_cell = np.full(n_targets, -1, dtype=np.intp)
    passive_indices: list[int] = []  # for non-delta-natal fall-through

    for k, i in enumerate(target_indices):
        natal_rid = int(pool.natal_reach_id[i])
        if natal_rid not in branch_rids_set:
            # Non-delta natal reach — fall through to passive tag.
            passive_indices.append(i)
            staged_rid[k] = -2  # sentinel: skip in vectorised commit
            continue

        # Per-agent dispatch: hatchery uses overridden p_home, wild
        # uses the species_config baseline. The hd-is-None-but-
        # ORIGIN_HATCHERY case (only reachable via SwitchPopulationEvent
        # cross-population transfer with mismatched hatchery configs)
        # logs a warning + falls back to wild precision rather than
        # raising mid-step. Multi-population alignment is a deferred
        # fix.
        if pool.origin[i] == ORIGIN_HATCHERY:
            if hd is None:
                logging.getLogger(__name__).warning(
                    "%s: ORIGIN_HATCHERY agent %d at homing time but "
                    "hatchery_dispatch is None — falling back to wild "
                    "homing precision. Likely SwitchPopulationEvent "
                    "crossed populations with mismatched hatchery "
                    "configs.",
                    ERR_HOMING_HATCHERY_NO_DISPATCH,
                    int(i),
                )
                p_home = species_cfg.wild.homing_precision
            else:
                p_home = hd.params.homing_precision
        else:
            p_home = species_cfg.wild.homing_precision

        # Build per-natal probability vector. Stray weights are
        # BRANCH_FRACTIONS over non-natal branches, renormalised.
        non_natal = [b for b in branch_rids_set if b != natal_rid]
        stray_w = np.array(
            [branch_stray_weight(b, mesh) for b in non_natal],
            dtype=np.float64,
        )
        stray_w /= stray_w.sum()
        all_branches = np.array([natal_rid] + non_natal, dtype=np.int8)
        all_probs = np.concatenate([[p_home], (1 - p_home) * stray_w])

        drawn_rid = int(rng.choice(all_branches, p=all_probs))
        staged_rid[k] = drawn_rid

        if drawn_rid != int(cur_reach[i]):
            entry_cell = _branch_entry_cell(mesh, drawn_rid)
            if entry_cell < 0:
                # Branch has no cells — log + leave staged_cell at -1
                # (no teleport). exit_branch_id is still set to the
                # drawn branch, but the agent stays in cur_reach. This
                # is desync, but the alternative (raise mid-step)
                # corrupts state. Documented limitation.
                logging.getLogger(__name__).warning(
                    "%s: branch_rid %d has no cells on mesh; "
                    "agent %d homing draw committed to "
                    "exit_branch_id but tri_idx not teleported.",
                    ERR_HOMING_BRANCH_ENTRY_MISSING,
                    drawn_rid,
                    int(i),
                )
            else:
                staged_cell[k] = entry_cell
        # else: drawn == current — no teleport (test 9).

    # Vectorised commit (atomic per-agent across the batch).
    homing_mask = staged_rid >= 0  # exclude sentinel -2 (passive)
    homing_idx = target_indices[homing_mask]
    pool.exit_branch_id[homing_idx] = staged_rid[homing_mask]
    teleport_mask = staged_cell >= 0
    pool.tri_idx[target_indices[teleport_mask]] = staged_cell[teleport_mask]
    if passive_indices:
        passive_arr = np.array(passive_indices, dtype=np.intp)
```

## Risk + regression surface

**Behaviour change for pre-C3.3 scenarios: zero.**
- YAMLs without `homing_precision` → dataclass default 0.95.
- `landscape=None` default + `is_baltic=False` short-circuit →
  passive-tag (existing behavior).
- All 6 existing `update_exit_branch_id` tests pass via fall-through
  AFTER `_FakePool` gains `natal_reach_id` and `origin` attrs.

**Behaviour change for C3.3-aware scenarios: by design.**
- Wild agents in delta branch → 95% return to natal branch.
- Hatchery agents → 65% return; 35% stray weighted by
  `BRANCH_FRACTIONS` over non-natal branches.
- `exit_branch_id` column in OutputLogger CSV reflects C3.3's
  stochastic decision (was passive observation pre-C3.3).
  Documented in agents.py field comment + `scientific-foundations-
  hatchery-wild.md` §7.
